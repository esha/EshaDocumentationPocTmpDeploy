AggregateVersion Class

Namespace: Esha.DomainModel
Assembly: Esha.DomainModel (in Esha.DomainModel.dll)
An AggregateVersion is an implementation of a compound identifier. It tracks the state of an entity, and aggregate entities can use the value of an AggregateVersion of children to determine if some part of its hierarchy has changed. This is useful for detecting if data that depends on the state of its children needs to be recomputed, or if that data can be used from a cached source. The action of values of this type is similar to the use of the _version field in C#'s List{T} (see: https://github.com/microsoft/referencesource/blob/5697c29004a34d80acdaf5742d7e699022c64ecd/mscorlib/system/collections/generic/list.cs#L43). This field is checked when enumerating the items in a List and an exception is thrown if the value is not the same as when enumeration started. Adds, removes and other list mutations will change this _version value, so enumerations can detect it.
